# 小龙虾用户独立表 CRUD 设计

## 1. 新表设计

表名：

```sql
operator_portal.follower_users
```

建表 SQL：

```sql
CREATE TABLE operator_portal.follower_users (
    id bigint GENERATED BY DEFAULT AS IDENTITY PRIMARY KEY,
    user_id character varying(64) NOT NULL,
    account_status character varying(24) NOT NULL DEFAULT 'active',
    strategy_status character varying(24) NOT NULL DEFAULT 'disabled',
    binding_status character varying(24) NOT NULL DEFAULT 'unbound',
    responsibility_domain character varying(32) NOT NULL DEFAULT 'unknown',
    created_at timestamp with time zone NOT NULL DEFAULT CURRENT_TIMESTAMP,
    created_by character varying(64) NOT NULL DEFAULT 'system',
    updated_at timestamp with time zone NOT NULL DEFAULT CURRENT_TIMESTAMP,
    updated_by character varying(64) NOT NULL DEFAULT 'system',
    is_deleted boolean NOT NULL DEFAULT false,
    deleted_at timestamp with time zone NULL,
    deleted_by character varying(64) NULL,
    row_version integer NOT NULL DEFAULT 1,
    CONSTRAINT ck_follower_users_account_status
        CHECK (account_status IN ('active', 'disabled')),
    CONSTRAINT ck_follower_users_strategy_status
        CHECK (strategy_status IN ('enabled', 'disabled')),
    CONSTRAINT ck_follower_users_binding_status
        CHECK (binding_status IN ('pending', 'bound', 'unbound')),
    CONSTRAINT ck_follower_users_responsibility_domain
        CHECK (char_length(btrim(responsibility_domain)) > 0),
    CONSTRAINT ck_follower_users_row_version
        CHECK (row_version > 0)
);

CREATE UNIQUE INDEX uk_follower_users_user_id_active
    ON operator_portal.follower_users (user_id)
    WHERE is_deleted = false;

CREATE INDEX ix_follower_users_account_status
    ON operator_portal.follower_users (account_status);

CREATE INDEX ix_follower_users_strategy_status
    ON operator_portal.follower_users (strategy_status);

CREATE INDEX ix_follower_users_binding_status
    ON operator_portal.follower_users (binding_status);

CREATE INDEX ix_follower_users_responsibility_domain
    ON operator_portal.follower_users (responsibility_domain);

CREATE INDEX ix_follower_users_updated_at
    ON operator_portal.follower_users (updated_at DESC);
```

字段说明：

| 字段名 | 类型 | 说明 |
| --- | --- | --- |
| `id` | `bigint` | 主键 ID |
| `user_id` | `varchar(64)` | 用户 ID，例如 `U-1201` |
| `account_status` | `varchar(24)` | 账户状态，值为 `active` / `disabled` |
| `strategy_status` | `varchar(24)` | 策略状态，值为 `enabled` / `disabled` |
| `binding_status` | `varchar(24)` | 绑定状态，值为 `pending` / `bound` / `unbound` |
| `responsibility_domain` | `varchar(32)` | 责任域，例如 `risk` |
| `created_at` | `timestamptz` | 创建时间 |
| `created_by` | `varchar(64)` | 创建人 |
| `updated_at` | `timestamptz` | 更新时间 |
| `updated_by` | `varchar(64)` | 更新人 |
| `is_deleted` | `boolean` | 逻辑删除标记 |
| `deleted_at` | `timestamptz` | 删除时间 |
| `deleted_by` | `varchar(64)` | 删除人 |
| `row_version` | `integer` | 乐观锁版本号 |
