# 质量规范

> 前端提交前至少要过 `lint`、`typecheck`，并保持中文文档与注释完整。

---

## 必须通过的检查

- `pnpm lint`
- `pnpm typecheck`
- 至少一次本地启动或构建级验证
- 文档、说明、模板、注释全部为中文

## 必须模式

- 每个函数写职责注释，说明用途、关键输入输出与必要副作用
- 每个变量都要有对应注释，或由紧邻的总注释统一说明
- 新增配置项必须说明默认值、用途和约束
- 页面必须有加载态、空态、错误态中的至少一种考虑；后台 CRUD 页面通常三者都要有
- `v0` 生成代码必须二次整理，不直接整段提交

## 禁止模式

- 页面直接请求后端地址，绕开 `/api/admin/*`
- 组件、配置或代理层出现英文主文案
- 为了赶进度省略注释
- 复制粘贴一整块 `v0` 或模板代码却不拆组件、不改命名
- 未经验证就把临时 mock 逻辑混入真实页面

## 测试与验证要求

- 骨架阶段至少验证首页可正常渲染
- 代理层至少验证成功转发、后端不可达、异常返回三类场景
- 新增页面后，至少覆盖列表、详情、创建或编辑、状态变更中的实际涉及场景

## 评审清单

- 目录落位是否符合 `app / components / features / lib / styles` 分层
- 是否有重复的卡片、表格、详情结构没有沉淀为组件
- 是否补齐了中文注释和配置说明
- 是否存在 `any`、双重断言、硬编码后端地址
- `v0` 产物是否已拆成可维护的仓库结构
