# 搭建 Go 后端基础框架

## Goal
在 `backend/` 目录下初始化一个可运行的 Go 后端基础框架，统一接入 Gin、Gorm、Sonic、Viper、Zap 和 Lumberjack，并提供最小可用的启动入口、配置体系、日志体系与数据库连接能力。

## Requirements
- 在 `backend/` 下初始化 Go Module，并完成基础目录结构搭建。
- 集成 `gin` 作为 HTTP 框架，提供基础路由与健康检查接口。
- 集成 `gorm` 与 PostgreSQL 驱动，提供数据库初始化与连接校验能力。
- 集成 `sonic` 作为 JSON 编解码能力，至少用于统一 JSON 响应输出。
- 集成 `viper` 作为配置加载组件，支持从配置文件和环境变量读取服务配置。
- 集成 `zap` 与 `lumberjack.v2`，提供结构化日志与文件切割能力。
- 提供默认配置文件和启动说明，确保本地可以快速启动。

## Acceptance Criteria
- [ ] `backend/` 下存在可读性清晰的 Go 项目目录结构。
- [ ] 执行 `go test ./...` 与 `go build ./...` 时可以通过。
- [ ] HTTP 服务可启动，并至少暴露一个健康检查接口。
- [ ] 配置文件可以控制服务端口、数据库连接与日志输出。
- [ ] 日志同时支持控制台输出与按文件滚动写入。
- [ ] 数据库初始化代码已接入 PostgreSQL，并包含基础连通性校验。
- [ ] 使用说明明确描述启动方式、配置项与依赖框架职责。

## Technical Notes
- 当前仓库暂无现成 Go 后端代码，本次实现会建立首套基础约定。
- 现有仓库根目录已存在 PostgreSQL Docker Compose 配置，可将其作为默认本地数据库依赖。
- 按项目要求，所有新增文档、函数注释和变量注释统一使用中文。
