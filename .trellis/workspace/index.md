# 工作区索引

> 记录所有开发者与 AI Agent 的工作记录

---

## 概览

这个目录用于跟踪本项目中所有开发者与 AI Agent 的协作记录。

### 文件结构

```
workspace/
|-- index.md              # 本文件 - 主索引
+-- {developer}/          # 每位开发者的目录
    |-- index.md          # 个人索引，包含会话历史
    |-- tasks/            # 任务文件
    |   |-- *.json        # 活动任务
    |   +-- archive/      # 按月份归档的任务
    +-- journal-N.md      # 日志文件（顺序编号：1、2、3...）
```

## 开始使用

### 新开发者

运行初始化脚本：

```bash
python3 ./.trellis/scripts/init_developer.py <your-name>
```

这会：
1. 创建你的身份文件（gitignored）
2. 创建你的进度目录
3. 创建你的个人索引
4. 创建初始日志文件

### 已存在的开发者

1. 先获取你的开发者名称：
   ```bash
   python3 ./.trellis/scripts/get_developer.py
   ```

2. 再查看你的个人索引：
   ```bash
   cat .trellis/workspace/$(python3 ./.trellis/scripts/get_developer.py)/index.md
   ```

## 规范

### 日志文件规则

- 每个日志文件**最多 2000 行**
- 达到上限后，新建 `journal-{N+1}.md`
- 创建新日志文件时，同时更新个人 `index.md`

### 会话记录格式

每次会话应包含：
- 摘要：一句话说明本次工作
- 主要变更：改了什么
- Git 提交：提交哈希与信息
- 后续事项：接下来该做什么

---

## 会话模板

记录会话时，请使用这个模板：

```markdown
## 会话 {N}: {标题}

**日期**: YYYY-MM-DD
**任务**: {task-name}

### 摘要

{一句话总结}

### 主要变更

- {变更 1}
- {变更 2}

### Git 提交

| 哈希 | 信息 |
|------|------|
| `abc1234` | {commit message} |

### 测试

- [OK] {测试结果}

### 状态

[OK] **已完成** / # **进行中** / [P] **阻塞**

### 后续事项

- {后续 1}
- {后续 2}
```
